Key-Value Databases

Key-Value Databases are basically huge buckets within which you store data that may or may not be related. A good way to envision Key-Value databases is to think of a locker room at a fitness facility. Each locker has an individual lock (the keys), and each locker has items stored inside of it (the values). Key characteristics are:

Key-Value Characteristics

  • The databases contain multiple key-value pairs
  • Each key-value pair can contain many different types of data. Using the locker room analogy above, locker #321 may contain sweatpants and protein powder while locker #456 contains a yoga mat and ballet shoes
  • Key-value pairs are stored in a similar way to the hash tables and maps used in popular programming languages

Popular Key-Value Implementations

Some of the top key-value implementations are:
  • Riak
  • memcached
  • Redis
