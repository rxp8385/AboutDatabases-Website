Document-Oriented Databases

Document-Oriented databases are really popular in the Big Data community. Like some of the other databases, they are designed to handle large datasets (sometimes comprising many terabytes of data), and there are tons of open source tools available to use on Document-Oriented databases. Some of their most important characteristics are:

Document-Oriented Characteristics

  • Data are stored in documents, which can make these databases easier to grasp for individuals new to Big Data tools
  • Documents can contain nested structures (meaning documents can contain other documents)
  • There is no need to create a pre-defined data model (or schema) prior to creating a Document-Oriented database. This makes them incredibly flexible and scalable

Popular Document-Oriented Implementations

There are two major open-source Document-Oriented implementations:
  • MongoDB
  • CouchDB
